# ag-linden

I build practical software solutions, with a current focus on [CodeRisk.ai](https://coderisk.ai).

CodeRisk.ai is an independent review practice for AI-powered products. It looks for hidden trust-boundary, state, consistency, integration, and decision risks that normal tests and happy-path demos often miss.

My work usually sits around:

- finding real defects in open-source systems
- turning risk patterns into small, durable fixes
- writing case files that explain why the issue was easy to miss
- improving software behavior at the boundary between users, agents, workers, data, and external systems

I care about fixes that are narrow, verifiable, and useful to maintainers.
